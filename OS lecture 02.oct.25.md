## Сети
### применение
- задача - распределить нагрузку на несколько машин. распараллелить
- *MPI* - message passing interface
- в кластере задачи разделяются на взаимодействующие (при помощи mpi) процессы
- кластеры по прежнему являются топовыми и работают на mpi или его производных
### нюансы сетей
- есть локальные терминальные устройства
- мультиплексоры позволяют экономить аппаратные порты (они дорогие)
- модем - устройство модуляции/демодуляции (цифра-аналог)
### модели сетей
#### Организация каналов
- каналы
	- постоянная связь в выделенном канале
- сообщения
	- неопределённый размер
- пакеты
	- заголовок + тело
	- фиксированный размер
	- произвольный порядок передачи
	- контрольные суммы для проверки целостности
	- при ошибке пакет отправляется снова
	- *нагрузка на сеть выше* из-за пакетов с уведомлениями (пакет получил/нет и т.п.) (вплоть до увеличения трафика на 900%!)
> активно развилось в 60-е годы при подготовке к войне
- ракетные шахты распределены по всей стране
	- этим нужно как-то управлять
	- связали компьютеризированными сетями
	- их нужно было обеспечить впечатляющей надёжностью
	- даже при повреждении части сети она должна была продолжить работу (в недетерминированных условиях качества среды передачи)
