# По курсу
- Понятие ресурса компьютера
	- Правила программного использования. (при их наличии можно рассматривать машину как ресурс, а не просто аппаратный компонент)
- Виртуальность - определение и понятие. (то, что функционирует в ВС и целиком/частично реализовано программно)
	- Виртуальные ресурсы с аппаратными аналогами (виртуальный принтер, виртуальная клавиатура и тп.)
	- Виртуальные ресурсы без аппаратных аналогов (ОС и т.п.)
- Работаем с ПО + аппаратурой. Не только лишь с железом.
- ВС - многослойный пирог с повышающимся по слоям уровнем абстракции. 
- В ВС множество компонентов, работающих вместе. У каждого компонента есть своя производительность.
- Производительность системы может определяться производительностью самого медленного компонента.
	- Процессор самый быстрый. Часто, остальное резко медленнее.
	- Зачем мир стремится делать быстрые процессоры, даже при медленной ОП? - История про сглаживание разницы.
- Сглаживание разницы:
	- Регистры общего назначения. (делается программистом или системой программирования + не решает всех проблем)
	- Кэш.
	- Расслоение памяти. (вместе с кэшем достаточно хорошо сглаживают дизбаланс).
- Проблема сглаживания дизбаланса скорости работы накопителя и ОП (дизбаланс колоссальный):
	- На ранних версиях машин обмен вызывался синхронно. (очень долго, а тогда ещё и дорого)
	- Асинхронность. (подаём заявку на обмен и занимаемся остальными делами, пока не прилетит прерывание по поводу)
- В ходе развития аппаратуры появились компененты, нужные для организации ПО и поддержки операционных систем.
	- Мультипрограммирование и мультипроцессирование (параллельно обрабатывается несколько процессов) (какие проблемы в таком случае появляются и какие аппаратные средства нужны для поддержания такой работы)
		- Исполнение.
		- Ожидание обмена.
		- Ожидание, пока планировщик ОС выставит статус исполнения.
	- Средства:
		- Разделение по правам: системный / пользовательский.
		- Аппарат защиты памяти (адрессное пространство процесса защищено от других).
		- Минимальные средства прерывания (хотя бы прерывание по таймеру).
- Аппарат виртуальной памяти:
	- Гибкость в сравнении с использованием исключительно физического адресного пространства.
	- Тема благодатная. Рассказать можно много.
	- Базирование, страничная, сегментная, сегментно-страничная организация и т.п.
- Страничная организация. (со всеми нюансами)
> Дальше просто нужно знать материал и уметь его излагать. Тех. часть.
- Пояснять разницу между ОС
	- Пакетная (б.б. квант, который может быть и не гигантским вовсе (меньше секунды). просто минимизирует количество переключений) / разделение времени 
- Нити - попробовать, прочувствовать.
- Взаимодействие процессов:
	- Разделяемый ресурс
	- Критические секции
	- Тупики, Deadlock и т.п. с примерами.
- Разбираться в нюансах приведённых в пример на синхронизацию программ.
- Семафоры Дейкстры - абстракция. Это не мьютексы. Это не семафоры IPC. (супермаркет с тележками и предельным количеством одновременно закупающихся пользователей)
- Модельные примеры:
	- Обедающие философы (равноправный доступ)
	- Читатели и писатели (резервирование)
	- Спящий парикмакер (клиент - сервер)

		- Демонстрировать и обсуждать работу с блоками и с индексными дескрипторами.

	- В остальном момент больше технический.
